#### 搭建一个有git记录的git项目仓库，并上传文件到远程仓库(可见的git服务器工作目录)

**第一步：登录远程服务器，建立仓库**

```
1. ssh [服务器ip地址] // 远程连接服务器

2. cd /home/git/ // 进入git总仓库

3. mkdir [项目名称] // 创建项目文件夹

4. cd [项目名称]

5. git init --bare [项目名称].git // 初始化空的git仓库

6. vim [项目名称].git/hooks/post-receive //编写以下内容

WORK_TREE='/home/git/[项目名称]/work'
git --work-tree="${WORK_TREE}" reset --hard

7. chmod u+x [项目名称].git/hooks/post-receive

8. mkdir work

9.cd ..

10. chown -R git:git [项目名称] // 更改项目文件夹的所属用户为git用户
```

**第二步：本地建立仓库，并上传项目到远程仓库**

```
1. git init // 有本地git仓库这一步可以省略

2. git add .

3. git commit -m "[评论内容]"

4. git remote add origin git@[服务器ip地址]:/home/git/[项目名称]/[项目名称].git // 准备到远程仓库

5. git push -u origin master // 上传远程仓库(第一次上传使用完成，后面都是git push)
```

#### 搭建一个空的git项目仓库，并上传文件到仓库(不可见的git服务器工作目录)

**第一步：登录远程服务器，建立仓库**

```
1. ssh [服务器ip地址] // 远程连接服务器

2. cd /home/git/ // 进入git总仓库

3. mkdir [项目名称] // 创建项目文件夹

4. cd [项目名称]

5. git init --bare // 初始化空的git仓库

6.cd ..

7. chown -R git:git [项目名称] // 更改项目文件夹的所属用户为git用户
```

**第二步：本地建立仓库，并上传项目到远程仓库**

```
1. git clone git@[服务器ip地址]:/home/git/[项目名称] // 克隆远程仓库

2. cd [项目名称]

3. 放置上传的内容到当前目录

4. git add .

5. git commit -m "[描述信息]"

6. git push -u origin master // 上传(master分支)远程仓库
```

备注：服务器上查不到项目的工作目录（git init --bare）

创建一个裸仓库，裸仓库没有工作区，因为服务器上的Git仓库只保存git历史提交的版本信息，纯粹是为了共享，所以不让用户直接登录到服务器上去改工作。
