今日结果：
     1  实体之间的关系（has- a)  , 继承之间的关系（is - a),; 继承分为单继承：一个子类只继承一个父类；多继承：一个子类可以继承多个父类。
     2  派生类的功能：可以继承基类的属性和行为（部分），并且可以自定义属性和行为；作用：加快程序的执行速度，减少程序的错误； 使用规则：若A类和B类是不相关的不能继承，类B是类A的“一种" 时可以继承。（class     B : class A , class C   {  }).
     3  友元函数属于外部函数，但是类的朋友在类中声明（friend  void  print(  )   ;).  在友元函数中可以实例化调用所有类的成员；  友元函数尽量少用， 其在类中的声明无访问权限，默认的继承方式为私有。 ”不可直接访问“  可以通过友元函数访问或成员函数访问。
     4   派生类不能继承基类的构造函数，只能间接调用； 初始化基类的构造函数（也指定调用父类构造函数）： 
   GRANDSON :: GRANDSON( Int age , char\* name): SON(1) {      }  ; 构造函数的调用顺序由上到下，析构函数与其相反；  在派生类中重定义基类的函数，之后派生类就直接调用自己的重定义函数，不去调用基类的函数；  重定义的函数与基类的原型函数形同；  调用基类的重定义函数； 重写函数调用， FATHER:: pub\_func();  主函数中调用  s. FATHER:: pub\_func();  
     5  指定调用父类的构造函数：  SON ::SON (int age ): FATHER("abc")  {......}  ； 基类的静态变量，是基类和派生类共用存储空间；并且没有所谓的继承的关系，只是”传家宝“的概念。
    6  友元类： friend  class FATHER ; A是B的朋友，C是B的儿子，  A可以访问B 的所有成员，也可以访问C从B中继承的成员，不能访问C的自己的成员。
   7  复合 与继承 ：讲的较少，可以自己去查阅资料。  
   8  类的虚拟继承：  class  A :　virtual public B , virtual  C  { ...}// 每一个继承类的前面都加 virtual 关键字；    作用： 解决继承过程中重复继承的问题。
   9  当重复继承的权限不同时，按保护级别级较低的继承。
   10  虚拟继承自动获取最近的基类的重写的成员方法：  class A : virtual  public  B (就近重写）,  virtual .......  ；  虚基类只会调用一次（不管是间接调用多少次或直接调用多少次）。
