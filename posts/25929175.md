今日结果：
    1 为了实现成员变量共享，也可以在类的外面定义成员变量；static 成员变量不能由构造函数初始
化，而是由类直接初始化 : MAN m1;int m1::cnt=9;  为静态的cnt初始化；由类访问访问静态成员变量
：m1::cnt.
    2 静态成员函数不能用const修饰，它没由this指针，只能访问静态成员的数据或函数，不能访问非
静态成员； 访问静态成员用类调用：MAN::Show();或m1.Show().
    3 为类分配内存空间，并且释放：MAN \*p = new MAN;  p->show();  delete p; .
    4 构造函数的特点：函数名与类名相同； 系统自动调用；无返回值；只能声明或定义在public中；
    5 带参的构造函数的调用（实例化对象数组）：MAN m[2] = {MAN(2), MAN(5)};
    6 const成员函数不能调用普通的成员函数（但可以调用static的 或 没有修改数据的函数）；类中
const 成员变量的赋值： const int cnt; const int num;
                       MAN ::MAN():cnt(10),num(20){....};
    7 常量函数的声明： int getname()  const;  它只能访问数据，但不能更改数据； 它外部的定义
：  int MAN::getname()  const  {....}.
    8 mutable 修饰符作用：存取权限的补充； mutable char name[10];加关键字之后在常函数下可以
修改其值。
    9 析构函数：
            普通的析构函数：有一个或多个参数。
            默认构造函数：没有参数，系统默认产生并初始化。
            复制（拷贝）构造函数：浅复制：将对象的数据成员值，进行简单的复制。深复制：在钱复
制的基础上，为指针类型变量生成新的空间。
      其功能：是已经存在的对象来初始化一个新的本类的对象。
        MAN m("abc") ;
        MAN m1(m);      // 拷贝构造函数（只有一个参数，并且为给类对象的引用：MAN::MAN(MAN 
        const<<m1.getname();                             //   &m){...}.
    10 浅复制：this->name = name;  深复制： strcpy(this->name, name).
           MAN  mytest()    MAN 类也可以当做自定义类型，且与其它普通类型用法一样。
           {
               MAN m("iwjo");
               return m;
           }
    11 转换构造函数（不安全）： MAN m = "abc"; （有一个参数的构造函数才能这样做）
                    explicit MAN m("abc");  explicit 防止转化构造函数的方法的使用。
          转换函数的方法类似于：  string name = "acb";的赋值。
    12构造函数：当这个函数快要结束时，系统自动调用该函数。
        其特点：无返回值，没有参数， 不能被重载，与类名相同（不过函数名前面加" ~ "标记）；
                 声明： ~MAN();
