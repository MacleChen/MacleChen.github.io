[虚拟机](http://www.2cto.com/os/xuniji/)下的ubuntu12.04设置桥接网络

VM版本9.0.1 build-894247

ubuntu12.04

1.设置虚拟机

选择virtual network editor

VMnet0设置为桥接，bridged to 选择以太网。

dhcp不选。

2.设置ubuntu

3.设置主机，网络连接中的vm1，打上勾

如果没有安装，可以通过下面的方式进行安装,要确保你的网络上了桥接的协议如下所示（网上找的，没验证）：

(1).在"VMware Network Adapter VMnet1属性"中,点"安装"->"服务"->"添加..."->厂商中选择"VMware,Inc",网

络服务中选择"VMware Bridge Protocol 版本:4.0.0.3从磁盘安装..."->"浏览"->C:/Program Files/VMware/VMware Workstation/netbridge.inf(这里选你自己安装目录下的)->"打开"

(2)现在在"VMware Network Adapter VMnet1属性"中看到了"VMware Bridge Protocol",钩选它

4.ubuntu系统设置

/etc/network/interfaces中添加：权限注意下，下同

auto eth0

iface eth0 inet static

address 10.10.111.211            (和主机同一个网段)

netmask 255.255.255.0

gateway 10.10.111.254            （和主机一样）

保存退出。

/etc/resolv.conf中添加：（DNS）

search localdomain

nameserver 202.106.46.151    （和主机一样）每次重启系统都检查一下，好像是重启后就得重新添加(我的是)。  后面解决方案1.

保存退出。

重启网络服务：sudo /etc/init.d/networking restart

连不上，重启下主机，再看一下网络设置

解决方案1：

这个文件是由resol
[vc](http://www.2cto.com/kf/ware/vc/)onf程序进行动态生成的，请不要手动修改，修改的信息将会被覆盖。

创建一个文件/etc/resolvconf/resolv.conf.d/tail：

#sudo vim /etc/resolvconf/resolv.conf.d/tail

然后我们在这个文件里写入自己要添加的DNS服务器，格式与以前的/etc/resolv.conf文件一致：

nameserver 202.106.46.151

然后输入保存退出。接下来我们重启下resolvconf程序，让配置生效：

#sudo /etc/init.d/resolvconf restart

再去看看/etc/resolv.conf文件，自己添加的DNS服务器果然乖乖的写进去了！至此问题完美解决！
