### 环境说明

- 本地mac os x 14.5系统
- 服务器版本是ubuntu-14.04

### 服务器操作

#### 1. 在服务器系统里新增一个用户，登录名为git

```
sudo adduser git   //  增加git用户，接着就会让你输入密码及一些用户信息，一直回车就行
```

#### 2. 初始化git仓库

进入git目录，新建文件夹travel作为代码仓库

```
1. cd /home/git/ // 进入git总仓库
2. mkdir [项目名称] // 创建项目文件夹
3. cd [项目名称]
4. git init --bare [项目名称].git // //其中--bare参数是为用户创建一个裸仓库
5. vim [项目名称].git/hooks/post-receive //编写以下内容
WORK_TREE='/home/git/[项目名称]/work'
git --work-tree="${WORK_TREE}" reset --hard

6. chmod u+x [项目名称].git/hooks/post-receive
7. mkdir work
8.cd ..

9. chown -R git:git [项目名称] // 更改项目文件夹的所属用户为git用户
```

#### 3. 修改文件的所有者，将/home/git目录下的所有文件及文件夹归git用户所有,并修改目录权限chmod 755 /home/git，在服务器上依次键入下列命令

```
chown -R git:git /home/git
chmod 755 /home/git
chmod 700 .ssh
chmod 644 .ssh/authorized_keys
```

#### 4. 修改ssh配置文件,修改完之后重启ssh

```
vim /etc/ssh/sshd_config
service ssh restart
```

AuthorizedKeysFile这个配置项默认是被注释掉的，现在删掉前边的#就行, 如下

```
RSAAuthentication yes
PubkeyAuthentication yes
AuthorizedKeysFile  %h/.ssh/authorized_keys
```

### 客户端操作

#### 1. 安装git工具（客户端）

       1.1 linux系统

```
sudo apt-get update        // 先更新
sudo apt-get install git   //安装git
```

     1.2 mac 系统(一般都git工具)

```
brew install git
```

#### 2. 配置git环境

```
git config --global user.name "你的用户名"
git config --global user.email "你的注册邮箱"
```

#### 2. 直接clone项目（要输入git用户密码）（如果忘记密码，使用 sudo passwd [用户名]  去更改密码, 例如： sudo passwd git）

```
git clone git@www.foverle.com:/home/git/[项目名称]/[项目名称].git
```

####

#### 3. 不想每次都输入密码，则在本地的mac系统生成ssh key

生成id\_rsa, id\_rsa.pub文件

```
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

一直回车就行，之后就可以看到.ssh文件了，里面有两个文件

- id\_rsa:称为私钥
- id\_rsa.pub:称为公钥，我们需要把公钥里的信息存入到服务器的/home/git/.ssh/authorized\_keys文件中

authorized\_keys是专门用来存公钥的地方。

#### ​4. 将id\_rsa.pub添加到git服务器

```
// 在客户端
cat id_rsa.pub   // 在终端查看，并拷贝到粘贴板

// 在git服务器上
sudo -s     // 切换为root用户
vim /home/git/.ssh/authorized_keys   //拷贝到这个文件里 （每一行一个）
```

好了现在就完成了将本地公钥添加到服务器。

#### 6.  修改authorized\_keys文件的权限并重新ssh服务

```
sudo chmod 644 /home/git/.ssh/authorized_keys
service ssh restart
```

####

#### 7. 重新clone项目就可以不用输入密码了

```
git clone git@www.foverle.com:/home/git/[项目名称]/[项目名称].git
```

#### **9. 禁用git用户的shell登录**

出于安全考虑，创建的git用户不允许登录shell，可以通过编辑/etc/passed文件完成，找到类似下面这一行：

```
git:x:1001:1001::/home/git:/bin/bash
```

改为：

```
git:x:1001:1001::/home/git:/usr/bin/git-shell
```

这样，git用户可以正常通过ssh使用git，但无法登录shell，因为我们为git用户指定的git-shell每次一登录就自动退出。
