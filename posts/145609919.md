问题：

```
➜  lib ll /var/lib/docker
ls: /var/lib/docker: No such file or directory
➜  lib
```

解决办法：

1. 安装debain的映射虚拟系统（docker run -it --privileged --pid=host debian nsenter -t 1 -m -u -n -i sh）

```
➜  ~ docker run -it --privileged --pid=host debian nsenter -t 1 -m -u -n -i sh
Unable to find image 'debian:latest' locally
latest: Pulling from library/debian
a492eee5e559: Pull complete
Digest: sha256:4abf773f2a570e6873259c4e3ba16de6c6268fb571fd46ec80be7c67822823b3
Status: Downloaded newer image for debian:latest
/ # %                                                                                                  ➜  ~
```

2.在docker中可以看到虚拟系统正在运行

![](https://i-blog.csdnimg.cn/direct/7896d490f4dd4aa299edf08d0109ed1a.png)

3. 在终端中再次查询（如果未出现，在多次执行：docker run -it --privileged --pid=host debian nsenter -t 1 -m -u -n -i sh）

```
➜  com.docker.docker docker run -it --privileged --pid=host debian nsenter -t 1 -m -u -n -i sh
/ # ll /var/lib
sh: ll: not found
/ # ls /var/lib
cni                 docker              lost+found          nfs
containerd          kubeadm             machine-id          swap
desktop-containerd  kubelet-plugins     mutagen             wasm
/ #
```

直到能查询到docker文件夹就是ok了
