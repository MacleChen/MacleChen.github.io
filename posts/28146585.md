3  命名管道： FIFO   （非亲戚进程通信,但通信是须在同一台计算机上）
第一种 ：   mkfifo  filename    // 创建管道文件
第二种： int  mkfifo (...);    //  在程序中进程创建（ 管道文件默认阻塞）
打开管道文件： fd = open ( "../myfifo", O\_RDONLY);     //  .. 上一级的目录（不能用读写的方式打开（O\_RDWR)
4 linux 下的PV操作 ：  信号量   相当于系统上的变量（只能取正整数或零）
只有两种操作：等待（P)， 信号（V)
最简单的信号量是只能取 ’0‘和’1‘ 的变量
通用信号量是指可以取多个正整数值
P(SV):  如果 SV > 0 ;  就减去 1 （通常减 多少可以由自己去设置）  ,如果SV =0 ; 就挂起进程执行
V(SV):  如果 SV < 0 ;  如果有其他进程因等待sv变量而被挂起，就让它恢复执行；如果没有进程因等待sv变量而被挂起，就给它加1
信号集处理函数 ：  semctl (...);     // sem信号量   ctl  control的缩写
创建信号：    int semget( key\_t key, int num\_sems,  int sem\_flag); /信号量集的标识码，信号量的个数，设置权限
成功时就返回一个正整数（信号量集的标识码）
如：   semid = semget( 3456, 1  , 0666 | IPC\_CREAT);     
find  ./   -name   kernel     查找文件位置
在内核的文件目录下：    /proc/sys/kernel
cat sem

 5  通过shell  查询信号量：  ipcs ;   通过shell 删除信号量： ipcrm  -s    标识码
给信号量设置初始值：    semctl( int sem\_id ,  int sem\_num ,  int cmd , ...);  // 信号量集对应的标识码，信号量的编号， 第四个参数（联合体）的出现可依赖于第三个参数的选择
  取信号量集的标识码：
ret = semctl( semid , 0 ,  GETVAL);
也可以在另外的进程中获取标识码
删除信号量集：  ret = semctl( semid , 0 , IPC\_RMID);
