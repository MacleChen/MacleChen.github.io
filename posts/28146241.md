今日结果： （进程）
       1  程序：是存在磁盘中的可执行文件； 进程：（PID），正在背执行的程序（在内存中）。
             进程间的特性：互不干扰，并行性。
       2  （PCB）进程控制快   主要分为：  代码段，数据段（存储全局变量和静态变量等），堆栈段（存储临时变量等）
       init 进程：（PID = 1） 不会终止，由内核调用。
       3  获取进程标识：   
                 返回值为int        pid\_t getpid(void);       //  获得进程pid
                                           pid\_t getppid(void);      // 获得父进程ppid
                     int pid\_t fork (void);     //  开启子进程
                         返回 ：  子进程返回0， 父进程大于0，为-1时出错。      
                         不能确定是父进程先执行还是子进程先执行。
                进程有涉及到：信号控制设置，进程优先级，进程组号，资源限制，终端控制（关闭终端，父子进程都结束）
                子进程与父进程区别：     父进程设置的锁，子进程不继承； 各进程ID不同
                fork（）系统调用后，父子进程将交替执行；  父进程退出后，子进程的父进程将为init 进程；
                子进程推出，父进程还在，此时子进程并没有真正的退出，而变成僵死进程，消耗资源，直到父进程捕获到子进程的退出或父进程调用wait（）才真正结束子进程。
               在用fork函数创建子进程后，子进程往往要调用exec函数。
              execl("/bin/ls", "ls", "-l" , NULL);         //    执行命令函数
              execlp("ls", "ls" ,"-l",  NULL);          
        4   exit  与  \_exit  的区别
               exit的作用：直接使进程停止运行，清除其使用的内存空间，并清除其在内核中的数据结构；
               exit与\_exit函数不同，exit函数在调用exit系统之前要检查文件打开情况把文件缓冲区的内容写回文件中去。如调用printf（）函数。
        5  write(1 ,  "testes" ,  6);       // 0 标准输入 ，  1 标准输出 ，  2 标准捡错
                 wait 函数用于使父进程阻塞，直到子进程结束。(waitid，  waitpid)
                 创建多个进程： 将fork循环产生想要的子进程个数。
        6  守护进程（如init进程）：在后台运行不受终端控制的进程，与运行前的环境隔开。
                   每个进程都会属于一个进程组
                   守护进程的实现： 创建子进程， 然后父进程终止
                                                  if(pid = fork())   exit(0);
                  子进程会把 控制终端，登录会话和进程组从父进程继承下来。
         7  chdir( const char\*);         //  更改工作目录
                     setid();     //  使子进程成为会话组长和进程组长（与控制终端分离）
                     nohup   ./a.out  &   //  后台运行（用终端分离）
