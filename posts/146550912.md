前提条件：iphone手机越狱ios15+，cydia/Sileo中安装好ssh，ldid等相关的常用插件

备注：如何iphone是ios15以下的越狱机，可直接看11步

1. 安装MonkeyDev

```
sudo /bin/sh -c "$(curl -fsSL https://raw.githubusercontent.com/AloneMonkey/MonkeyDev/master/bin/md-install)"
```

2. 打开XCode，选择MonkeyDev下的Comand-line Tool

![](https://i-blog.csdnimg.cn/direct/77a30d4070754d2a8d7feaf4a0b9cdc0.png)

3. 可以选择，c，c++，OC（Foundation），的脚本

![](https://i-blog.csdnimg.cn/direct/ff6f0795342040e3b1e1b55e60a6ea6f.png)

Include Simple PreferenceLoader 选项是创建一个可以在系统设置中配置的条目

![](https://i-blog.csdnimg.cn/direct/5fff29fdbd8944c794c343b4b5cb8f60.png)

4. 以Foundation的OC为例，Next创建项目![](https://i-blog.csdnimg.cn/direct/c19103001a46497ba6581b890d105917.png)

5. Comand+B编译项目

6. 遇到编译问题: Received disconnect from ::1 port 22:2: Too many authentication failures

解决办法：配置手机的ip，密码，端口等信息

![](https://i-blog.csdnimg.cn/direct/5186015f3d304fe38e55d237e69810f3.png)

7. 遇到编译问题: ackage architecture (iphoneos-arm) does not match system (iphoneos-arm64)

解决办法：需要在control文件中配置成arm64

![](https://i-blog.csdnimg.cn/direct/0903fdbead51470b9c8abc89e7db4175.png)

8. 遇到编译问题: unable to create '/usr/bin/TaskDaemonOCDev.dpkg-new' (while processing 'usr/bin/TaskDaemonOCDev'): Read-only file system

原因分析：ios15以下的越狱机都是rootful有根的越狱，可以完全访问整个文件系统，ios15以上的越狱机都是rootless有根的越狱，只能访问部分文件系统，都需要放在/var/jb/目录下，才有读写权限。

解决办法：修改Installation directory目录放在/var/jb下

![](https://i-blog.csdnimg.cn/direct/ce565b9fbedd4f9b9f9850fbc2d4fb58.png)

还需要将工程中的usr文件夹放在var/jb/目录下（var/jb目录需要手动创建）

![](https://i-blog.csdnimg.cn/direct/9cd8b56f3ecb4738961222d612061447.png)

再次编译即可成功

![](https://i-blog.csdnimg.cn/direct/b14649c04fcf4db89aa9cd18ce7f5fa4.png)

9. 可以查看效果：在mac终端登录手机ssh，进入/var/jb/usr/bin, 查看ls -all TaskDaemonOCDev

![](https://i-blog.csdnimg.cn/direct/f13a578fc8864d2d92c9824aef676480.png)

![](https://i-blog.csdnimg.cn/direct/0018f926f5ff4976892b17ce1a81a4f5.png)

10. 手机上运行遇到的问题：zsh: killed     ./TaskDaemonOCDev （ios15系统一下是：killed 9）

原因分析：iOS 由于安全限制（如 AMFI 和代码签名），需要**关闭代码签名**或**使用正确的工具链**。

解决办法：ldid -S TaskDaemonOCDev

![](https://i-blog.csdnimg.cn/direct/133acd6ffc4b44c6b1ebbdb07f537b2f.png)

10.1 偶遇到问题

/Users/chenfan/Desktop/zejin/root\_daemon/DamonOCRun/dpkg:1:1: parsing file '/var/jb/Library/dpkg/updates/0000' near line 13 package 'com.yishuihuayuan.damonocrun':

解决方法：这个错误通常是由于 **`dpkg` 配置损坏** 或 **`updates` 目录中的文件格式错误**。

移除坏掉的文件：rm -rf /var/jb/Library/dpkg/updates/\*，再次运行即可

![](https://i-blog.csdnimg.cn/direct/8ad6b9adb3444b0ba4e9ef60195df594.png)

![](https://i-blog.csdnimg.cn/direct/9d839d469872402e9718922f4d8972a3.png)

大功告成，以上算是运行成功了！
