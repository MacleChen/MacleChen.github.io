### git命令的相关操作

    a. 撤销本地的修改
        git checkout .   // 使用 git diff . 查看修改点
    b. 撤销暂存区（git add .）的修改
        git reset --hard xxx    // 使用git reflog 查看本地修改log
    c. 撤销git commit 的修改
        git reset --hard commit\_id // 实用 git log 查看提交的版本记录
    d. 生成patch信息（备注，必须在修改文件的那个目录下执行，不然会报错，提示ba request）
        git format-patch 856be50ddfc78f0dbf628ca6da230c2d4c3921da -1o patch  // 生成执行的提交记录的patch （856be50ddfc78f0dbf628ca6da230c2d4c3921da）是在
        https://dev.azure.com/Ricoh-SOP-TS/SecurityPatch/\_git/platform\_system\_security/commit/6df5a00b67de211f5cacbb28d942fa57424a1fd1?refName=refs%2Fheads%2FTS-develop 如果没有找到patch记录，尝试切换分支。
        或git format-patch 856be50ddfc78f0dbf628ca6da230c2d4c3921da -1o --patch
    e. 强制合并
        git add . // 将自己的代码加入暂缓区
        git stash // 将暂缓区内容压栈
        git pull // 重新拉取代码
        git stash pop // 出栈合并代码

    f. 合并Master分支
        git push // 提交当前分支
        git checkout master // 切换到主分支
        git pull // 更新主分支
        git merge dev // 将dev分支合并到master分支
        如果出错可以忽略文件：git checkout [文件]
        git push -u origin master // 提交最新master分支到git远端
    g. 给Master分支打Tag
        git tag v1.0.2.45 -m "添加45版本"// 添加tag
        git tag // 查询当前分支的tag列表
        git push origin --tags // 提交所有tags
        【以下可选】
        git push origin v1.0.2.45
        提交所有本地分支到服务器（只提交指定分支：git push origin v1.2）
    h. 删除tag分支
        git tag -d v1.0.2.58  // 删除本地
        git push origin :refs/tags/v1.0.2.58  // 删除远端tag

    other: git reset HEAD^ 回退上一个版本
    
    h. 解决Git :fatal: refusing to merge unrelated histories
        git pull origin master --allow-unrelated-histories
        
    i. 创建分支，并提交本地代码到新分支上
        git branch [name] // 创建分支
        git branch // 查看分支
        git push origin [name] // 提交本地代码到新的远端分支上
        git checkout [name] // 切换分支
    j. 解决SourceTree中不停弹输入密码框问题
        git config credential.helper store
