为大家讲解[Ubuntu](https://www.linuxidc.com/topicnews.aspx?tid=2) 16.04下安装Tomcat 8.5.9的过程步骤，希望能对你有所帮助。

在Windows下，到[官网](http://tomcat.apache.org/download-80.cgi#8.5.9)下载tomcat8.5.9，选择格式为tar.gz，如下图所示： 
 　　 
 　　![image_1b6g9rtvkn6vpdj1fjnv8s17jd9.png-66.4kB](https://i-blog.csdnimg.cn/blog_migrate/aff13c54f539e1b7d3a705eed10fd984.png)
 　　 
 　　通过ftp将下载的tomcat8.5.9压缩包上传到ubuntu服务器，配置VSFTPD可以参看《[Ubuntu 16.04下vsftpd 安装配置实例](https://www.linuxidc.com/Linux/2017-06/144807.htm)》[http://www.linuxidc.com/Linux/2017-06/144807.htm](https://www.linuxidc.com/Linux/2017-06/144807.htm)，然后这里我使用的是Xftp 5作为windows下使用的FTP客户端。上传完成后，进入该文件目录，使用如下命令解压：

```
sudo tar -zxvf apache-tomcat-8.5.9.tar.gz
```

　　解压成功后可以看到该目录下多出了文件夹apache-tomcat-8.5.9： 
 　　 
 　　![image_1b6ga1g751e97obqsas1u2oardm.png-19.3kB](https://i-blog.csdnimg.cn/blog_migrate/561ad0f271416a0bca11ca2bdef5c9a3.png)
 　　 
 　　先在/usr下新建文件夹tomcat，然后将文件夹apache-tomcat-8.5.9移动到目录/usr/tomcat下：

```
sudo mv apache-tomcat-8.5.9 /usr/tomcat/
```

　　进入目录/usr/tomcat可以查看到该文件夹，说明移动成功： 
 　　 
 　　![image_1b6ga580m1rig16oqls1frj1v4n13.png-23.6kB](https://i-blog.csdnimg.cn/blog_migrate/74695a539b7b68f4f96c401dc36862bc.png) 
 　　 
 　　现在先修改一下tomcat文件夹的使用权限，否则可能在当前用户下不能进入bin目录：

```
cd /usr
sudo chmod 755 -R tomcat
```

![image_1b6ga9ol51un67oocan1fgelj1g.png-90kB](https://i-blog.csdnimg.cn/blog_migrate/e37fe81a6b38086db72448a41415907a.png) 
 　　 
 　　然后进入目录/usr/tomcat/apache-tomcat-8.5.9/bin，编辑文件startup.sh，在最后一行之前加入如下信息：

```
#set java environment
export JAVA_HOME=/usr/java/jdk1.8.0_111
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:%{JAVA_HOME}/lib:%{JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH

#tomcat
export TOMCAT_HOME=/usr/tomcat/apache-tomcat-8.5.9
```

　　即： 
 　　 
 　　![image_1b6gagcio194uimi1fs5119d2f1t.png-99.2kB](https://i-blog.csdnimg.cn/blog_migrate/ca6308d942a3bdbe58d0cae862bf1f7c.png)
 　　 
 　　其中JAVA\_HOME和TOMCAT\_HOME请对应你自己的jdk和tomcat的安装目录。编辑完后保存退出，然后运行startup.sh：

```
sudo ./startup.sh
```

　　出现如下信息则说明tomcat安装成功，并且已经启动。 
 　　 
 　　![image_1b6gavbrh1rp01t6bebh1lv71voe3h.png-55.3kB](https://i-blog.csdnimg.cn/blog_migrate/6c3deede7955ba071a1af2914bf3be9d.png)
 　　 
 　　而且可以在Windows下访问虚拟机的8080端口： 
 　　 
 　　![image_1b6galedl11ia1t841iom1ka9odb2n.png-264.1kB](https://i-blog.csdnimg.cn/blog_migrate/d1a1bbc8d72b8bcc177ba132c14321b3.png)

```
#set java environment
export JAVA_HOME=/usr/java/jdk1.8.0_111
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:%{JAVA_HOME}/lib:%{JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH

#tomcat
export TOMCAT_HOME=/usr/tomcat/apache-tomcat-8.5.9
```

　　然后执行如下命令即可：

```
sudo ./shutdown.sh
```

![image_1b6gb0ujfqlci641ucg1tih11v43u.png-61kB](https://i-blog.csdnimg.cn/blog_migrate/6fc65602a2a9fac8c94456274b4f1707.png)

　　如果要设置为tomcat开机自启动，需要编辑文件/etc/rc.local，这里存放着开机自启动的程序。（配置在/etc/profile和/etc/bash.bashrc文件中的内容是当有用户登录时才起作用，这不符合tomcat需要启动的实际情况） 
 　　现在我们编辑/etc/rc.local：

```
sudo vi /etc/rc.local
```

　　在最后一行之前加入如下信息：（配置你自己的tomcat的startup.sh文件的路径）

```
#set java environment
export JAVA_HOME=/usr/java/jdk1.8.0_111
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:%{JAVA_HOME}/lib:%{JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH

/usr/tomcat/apache-tomcat-8.5.9/bin/startup.sh
```

　　即： 
 　　 
 　　![image_1b6gcfe7c1jl5pb61bld1fl8klc4b.png-155.6kB](https://i-blog.csdnimg.cn/blog_migrate/c32f0b14beacd79eab301f720ba421f6.png)
 　　 
 　　执行命令reboot重启系统，然后即可通过在windows下访问虚拟机的8080端口验证tomcat服务已启动： 
 　　 
 　　![image_1b6gci8rad861pp8161j1nff9nv4o.png-277.2kB](https://i-blog.csdnimg.cn/blog_migrate/1a03c64d5e28d4bc31e33ce32f88b807.png)
