今日结果：（模版）
1 模版： 使用一种完全通用的方法，来设计函数或类
2 函数模版： 对任意类型的变量进行操作的函数
tempalte <模版形参表>   
返回值类型   函数名（模版函数形参表）
{ ... }
在函数的基础上对函数进行整合
template <typename T>    // typename 或class 效果相同
T getmax(Ta ,T b)｛        //模版与函数名之间不能有其他的语句
if(a > b)
return a;
return b;
}
泛型编程-》一般指函数模版，类模版
getmax<int>(1, 2) 或  getmax<double>(1, 2) // 强制转换类型
如果没有强制转换，则根据参数类型判断
函数模板也支持重载
3 类模版：  实际是函数模版的推广， 可以用相同的类模版来组建任何类型的对象集合
template <class T1, class T2>
class  MAN
{...};        // 类模版的中成员函数的定义与声明不同
 当类模版的声明与定义分开时，应该在有主函数的文件中加入  #include "\*\*\*\*.cpp" 函数定义的文件
4 派生类模版： （继承同样也可以产生派生类模版）
从一般类派生出类模版：
template <class T1, class T2>
class MAN : public PEOPLE
{...};   
从类模版派生出类模版：
template <class T>
class ABC : public AA<T> // 也可以是具体的类型
       ｛...｝;   // 继承类模版
类的二义性：子类可以调用自己的成员函数，也可以调用父类同名的成员函数。
默认调用自己的成员函数
MAN<Cchar\*,int> m ;   // 类模版实例化一个对象
 m.PEOPLE<char\*> :: getname(); // 子类调用父类的成员函数
