今日结果：线程的应用
1  线程： 一个进程中至少都有一个执行线程； （多线程：就是在一个程序中有多个执行路线）； “也叫做轻量级进程”；
也是一个进程中的控制序列。
2 fork,与创建线程的区别： 每个线程可以拥有自己的独立变量，多线程间可以共享全局变量，文件描述符等。
创建一个子进程的系统开销是一个线程的20多倍。
3  线程的优点：系统的开销比进程较小； 线程间的切换需要做的工程比进程较小（在内存上是这样，CPU,不一定）；
线程占用的资源要比进程少很多。（CPU核的概念： 核心数越多，多线程的数量就越多，执行效率就越快）
4  线程缺点：一个线程出错，可能其他线程也会出错。（在时间分配上和共享内存上）
5 线程函数库：<pthread.h>   在编译时需要加上-lpthread( g++ -o main main.cpp -lpthread)  (并非系统库）
每个线程也有一个ID,只在所属的进程环境中有效。
线程ID用 pthread\_t  类型表示
  int  pthread\_create(...) //  创建线程函数    成功返回0
pthread\_exit("abc");    //   结束线程函数        输出额外信息（可以写入处理的函数指针中）  返回给pthread\_join 的第二个参数
pthead\_join 相当于wait函数，等待回收资源；  例如： ret = pthread\_join(pt ,  NULL);  // 作用等待线程执行完毕
创建线程实例： pthread\_t pt;   ret = pthread\_create(&pt, NULL, funcl, NULL);    //   void\* funcl (void\* argv);
当进程存在时，才会有线程存在   
创建多线程可以同时执行同一个函数，可以共享全局变量
在pthread\_exit函数中返回值不能用一个函数里的局部变量。
堆： 手动申请空间，手动释放（相当与全局变量）
栈： 有系统自己申请，释放，属于临时存储空间（局部变量，函数调用）
6 线程同步：（使用信号量，互斥量，全局变量）
轻量级信号量（没有key值的概念）在线程里面用（<semaphore.h>)
信号量对象用 sem\_t 表示
信号量的创建与初始化：
int sem\_init(sem\_t \*sem, int pshared, unsigned value); 信号量初始化函数    信号量  0表示当前信号量   初始值
int sem\_post(...)  // 相当与真正信号量中的v 操作 （加1）
int sem\_wait(...)  // 想当于p操作（减1） （非零是才可以减）  
int sem\_destroy(...)  //  销毁信号量，回收资源
7  互斥量（“线程锁") 进行同步
互斥对象用  pthread\_mutex\_t 表示
pthread\_mutex\_lock(...)  // 加锁
pthread\_mutex\_unlock(...)  // 解锁
pthread\_mutex\_destroy(...)  // 销毁锁
pthread\_mutex\_init(...)   // 锁的初始化
线程里面的函数指针不能用类里面的函数，但可以将类的对象作为参数传递
8  条件变量的函数:
int  pthread\_cond\_init(...)  // 条件变量的很初始化
int pthread\_cond\_wait(...) //  等待条件函数    此函数的过程： 先解锁， 等待信号， 如果在
一定的世间内等到信号量就在解锁，否则等待失败，继续向下执行。
int pthread\_cond\_timewait(...) //时间等待函数
int pthread\_cond\_destroy(...)  // 销毁条件的函数
int pthread\_cond\_signal(...)  // 发送条件的函数
int pthread\_cond\_broadcast(...)  // 发送广播信号的函数
usleep(100);   // 休眠的作用是为了放置自己解锁后再加锁。
