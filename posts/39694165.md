在macbook pro上(操作系统为Lion)安装PyAMF，报了这个错误：

**unable to execute gcc-4.2: No such file or directory
error: command 'gcc-4.2' failed with exit status 1**

**错误分析：提示是找不到gcc-4.2，那么就从这里入手，看看为什么找不到，然后让系统找得到gcc。**

去/usr/bin看了一下，没有gcc，当然敲gcc命令也一样是没有。

确定没有gcc之后，可以自己安装gcc。我这里用一种比较简单的方法，去实现macos下的gcc安装：

首先安装Xcode。由于我用的是Xcode4.3.2，安装就是把Xcode拖拽到应用程序文件夹。

之后点开Xcode，然后左上角苹果边上的Xcode，选择Preferences --> Downloads。之后点击Command Line Tools (180 MB) 这行的 Install 按钮。

![](https://img-my.csdn.net/uploads/201205/31/1338458696_9968.png)

安装好之后，敲一下gcc，这次有了，显示：

i686-apple-darwin11-llvm-gcc-4.2: no input files

这次再尝试安装pyamf，一次通过了！

**总结一下，原因是macos 10.7.4不自带gcc。而xcode4.3.2也不带……这样，就造成没有gcc。**

**解决办法：通过xcode4.3.2的Downloads功能安装上Command Line Tools 间接安装上gcc。**

ps：如果还是不行的话，尝试一下

做连接gcc-4.2到命令llvm-gcc-4.2

sudo ln -s /usr/bin/llvm-gcc-4.2 /usr/bin/gcc-4.2
