问题6：

can't link a dylib with itself. same install\_name as dylib being built in

解决办法：

- 前往您 dylib 目标的 **Build Phases** (构建阶段) 选项卡。
- 检查 **Link Binary With Libraries** (链接二进制文件与库) 部分。
- 如果看到 dylib 文件本身（例如，`libMyLibrary.dylib`），**请将其移除**。

![](https://i-blog.csdnimg.cn/direct/f113409eda2e48babdf90cd7885a373e.png)

(废除)~~解决办法：修改动态库为静态库（备注：动态库也可以的，这个问题也可以忽略， 有些会有影响），需要注意的是：对于springboard来说改为静态库后有可能导致库挂载不上~~

![](https://i-blog.csdnimg.cn/direct/85dbdf12342e48c79ad960fa484d2ad8.png)

另外的解决办法（保证动态库）：删除目录下的dylib库，因为build他依然会存在

问题8：

error creating directory 'Library/MobileSubstrate': Read-only file system

原因分析：ios15以下为rootful有根目录，越狱后的设备都可以修改系统目录；ios15以上的系统任何越狱设备都是rootless无根目录，不能修改/Library目录，需要修改成/var/jb的目录下操作

解决办法：

        1. 将代码块的Library文件夹或usr或ect文件夹放在/var/jb目录下

![](https://i-blog.csdnimg.cn/direct/50e26676742441a3b675fa7af5fde54a.png)

        2. 修改installation directory的地址为/var/jb/Library/xxxx

![](https://i-blog.csdnimg.cn/direct/d5503a9483d642e59746cc4e08e85092.png)
