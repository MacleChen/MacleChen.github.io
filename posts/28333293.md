今日结果：（第四阶段 开发板的学习和使用）

1. uboot的用法（也叫bootloader）

主要依赖于硬件，基本上不肯能写成通用的文件。

ARM（一些是三星产），X86（intel CPU）

交叉编译：在这个编译环境中去编译另一个环境中的代码。

ARM ---》UBoot编译后， 生成boot.bin 二进制的可执行文件

2. arm板 920  （CPU）s3c2440(A)

存储设置： norflash: ox0000  ----  ox200000    （2M）有地址线，可以执行程序

nandflash：（相当于硬盘）没有地址线，可以执行程序。

内存：ox3000 0000  ---  ox3400 0000 有地址线，可以执行程序。

3. 往开发板上烧写uboot ，通过JLink， 注意：开发板上的拨号开关要拨到Norflash侧。

内存的烧写：  1，U盘烧写   2.网线    3.usb

sz：从ARM板上向PC机发送文件

rz：在ARM板上接收PC机传来的文件

4. 烧写文件系统：USB start

fatload usb  0  ox3000 8000  rootfs.img  -- 把U盘上的rootfs.img 烧写到开发板上。

cc  等同于 gcc

arm-linux-gcc main.c -o main (交叉编译) 在PC机上编译ARM板上的执行文件

arm-linux-g++ main.cpp -o main   同上
